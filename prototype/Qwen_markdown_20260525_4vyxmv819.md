# DanmuAI 温馨风格设计系统 (Design System)

## 1. 设计目标

DanmuAI 的设计目标是为主播提供一个**轻量、隐私友好、无压迫感**的 AI 弹幕助手。视觉上摒弃传统科技工具的“冰冷感”与“效率至上”的深色调，转而采用**温馨、治愈、放松**的咖啡馆氛围。通过柔和的色彩、圆润的几何形态和拟人化的交互反馈，降低主播在直播过程中的焦虑感，同时清晰传达“隐私保护”与“智能陪伴”的产品核心价值。

## 2. 视觉关键词

- **温馨治愈** (Warm & Healing)
- **圆润亲和** (Rounded & Friendly)
- **奶油质感** (Creamy Texture)
- **轻盈呼吸** (Light & Breathable)
- **隐私安全** (Safe & Private)

## 3. 色彩规范

色彩系统以低饱和度的暖色调为主，避免高对比度带来的视觉疲劳。

### 3.1 品牌色与基础色


| 变量命名 (CSS Variables) | Tailwind 扩展名   | 色值 (HEX)  | 用途说明                         |
| -------------------- | -------------- | --------- | ---------------------------- |
| `--color-primary`    | `warmPink`     | `#FFB6B9` | 品牌主色，用于主按钮、激活状态、核心图标。        |
| `--color-secondary`  | `softPeach`    | `#FFE5D9` | 品牌辅助色，用于侧边栏背景、次要标签、Hover 背景。 |
| `--color-accent`     | `pastelYellow` | `#FFF2CC` | 提示/高亮色，用于隐私小贴士、警告提示框、侧边栏边框。  |
| `--color-bg`         | `cream`        | `#FDFBF7` | 全局页面背景色，提供奶油般的柔和底色。          |
| `--color-surface`    | `white`        | `#FFFFFF` | 卡片、弹窗、输入框表面色。                |


### 3.2 文本色


| 变量命名                     | Tailwind 扩展名 | 色值 (HEX)  | 用途说明                     |
| ------------------------ | ------------ | --------- | ------------------------ |
| `--color-text-primary`   | `warmText`   | `#5D5757` | 主标题、正文、核心数据（替代纯黑，降低对比度）。 |
| `--color-text-secondary` | `gray-400`   | `#9CA3AF` | 副标题、占位符、辅助说明文字。          |


### 3.3 状态色 (语义色)


| 状态               | 色值 (HEX)        | 浅色背景 (用于 Alert/Badge) |
| ---------------- | --------------- | --------------------- |
| **成功 (Success)** | `#86EFAC` (柔和绿) | `#F0FDF4`             |
| **警告 (Warning)** | `#FDE047` (柔和黄) | `#FEFCE8`             |
| **错误 (Error)**   | `#FCA5A5` (柔和红) | `#FEF2F2`             |
| **信息 (Info)**    | `#93C5FD` (柔和蓝) | `#EFF6FF`             |


## 4. 字体规范

采用 **Nunito** 字体族，其圆润的字形特征与“温馨亲和”的视觉语言完美契合。


| 层级     | 字号 (px/rem)     | 字重 (Weight)    | Tailwind 类名             | 应用场景                      |
| ------ | --------------- | -------------- | ----------------------- | ------------------------- |
| **H1** | 24px / 1.5rem   | Bold (700)     | `text-2xl font-bold`    | 页面大标题 (如: 下午好，今天也要开心直播呀！) |
| **H2** | 20px / 1.25rem  | Bold (700)     | `text-xl font-bold`     | 区块标题 (如: 助手配置)            |
| **H3** | 18px / 1.125rem | Bold (700)     | `text-lg font-bold`     | 卡片标题、模态框标题                |
| **正文** | 16px / 1rem     | Regular (400)  | `text-base`             | 段落文本、列表内容                 |
| **标签** | 14px / 0.875rem | SemiBold (600) | `text-sm font-semibold` | 表单 Label、侧边栏导航项           |
| **辅助** | 12px / 0.75rem  | Regular (400)  | `text-xs`               | 隐私提示小字、时间戳、Badge 文本       |


## 5. 间距规范

基于 `4px` 基础网格，采用较为宽松的间距以营造“呼吸感”。

- **页面边距 (Page Padding)**: `32px` (`p-8`)
- **模块间距 (Section Gap)**: `32px` (`gap-8` / `mb-8`)
- **卡片内边距 (Card Padding)**: `24px` (`p-6`) 或 `32px` (`p-8`)
- **表单元素间距 (Form Gap)**: `16px` (`gap-4`) 或 `24px` (`gap-6`)
- **图标与文字间距**: `12px` (`gap-3`)

## 6. 圆角规范

**严禁使用尖锐直角**，所有元素必须经过圆角处理以传递安全感。


| 变量命名            | 数值             | Tailwind 类名    | 应用场景            |
| --------------- | -------------- | -------------- | --------------- |
| `--radius-lg`   | 24px (1.5rem)  | `rounded-3xl`  | 大卡片、模态框、全局弹窗    |
| `--radius-md`   | 12px (0.75rem) | `rounded-xl`   | 按钮、输入框、小卡片、导航项  |
| `--radius-sm`   | 8px (0.5rem)   | `rounded-lg`   | 内部标签、小图标容器、下拉菜单 |
| `--radius-full` | 9999px         | `rounded-full` | 头像、状态指示灯、胶囊按钮   |


## 7. 阴影规范

摒弃冷灰色的生硬阴影，采用带有品牌色调的**暖色弥散阴影**。

- **基础暖阴影 (Surface)**: `0 4px 20px rgba(255, 198, 198, 0.1)` (用于静态卡片)
- **悬停暖阴影 (Hover)**: `0 10px 25px rgba(255, 198, 198, 0.2)` (用于卡片 Hover 浮起)
- **强调阴影 (Primary)**: `0 4px 15px rgba(255, 182, 185, 0.4)` (用于主按钮点击/悬停)

## 8. 布局规范

- **全局架构**: 左右分栏布局 (Flexbox)。
- **侧边栏 (Sidebar)**: 固定宽度 `256px` (`w-64`)，背景纯白，右侧带有 `1px` 的 `pastelYellow` 边框。
- **主内容区 (Main)**: `flex-1`，背景 `cream`，内部采用 CSS Grid 进行栅格布局。
- **栅格系统**: 
  - 统计数据: `grid-cols-3`
  - 表单配置: `grid-cols-2`
  - 坐标输入: `grid-cols-4`

## 9. 按钮规范


| 类型                  | 样式特征                                                 | 交互反馈                                    |
| ------------------- | ---------------------------------------------------- | --------------------------------------- |
| **主按钮 (Primary)**   | 渐变背景 (`#FFD1D1` -> `#FFB6B9`)，白字，`rounded-xl`，带强调阴影。 | Hover: 渐变加深，阴影扩大；Active: `scale(0.98)`。 |
| **次按钮 (Secondary)** | 纯白背景，`1px` 灰色边框 (`border-gray-200`)，暖灰字。             | Hover: 背景变 `gray-50`。                   |
| **幽灵按钮 (Ghost)**    | 透明背景，无框，品牌色文字。                                       | Hover: 背景变为 `softPeach`。                |
| **危险按钮 (Danger)**   | 浅红背景 (`#FEF2F2`)，柔和红文字 (`#FCA5A5`)。                  | Hover: 背景加深。                            |
| **禁用状态**            | `opacity-50`, `cursor-not-allowed`，去除所有 Hover 动效。    | -                                       |


## 10. 表单组件规范

- **输入框 (Input)**: 
  - 默认: `bg-cream`，无边框 (或极浅边框)，`rounded-xl`，`py-3 px-4`。
  - Focus: 边框变为 `warmPink`，外发光 `box-shadow: 0 0 0 3px rgba(255, 182, 185, 0.2)`。
- **下拉选择 (Select)**: 样式同输入框，右侧带有自定义的柔和箭头图标。
- **复选/单选 (Checkbox/Radio)**: 选中时填充 `warmPink`，边框圆角 `4px` (Checkbox) 或 `full` (Radio)。
- **开关 (Switch)**: 轨道背景 `gray-200`，激活背景 `warmPink`，滑块纯白带暖阴影。

## 11. 卡片组件规范

- **基础容器**: `bg-white`, `rounded-3xl`, 基础暖阴影。
- **Hover 动效**: `transform: translateY(-4px)`，过渡时间 `0.3s cubic-bezier(0.4, 0, 0.2, 1)`。
- **统计卡片**: 左侧 `48x48` 圆角 (`rounded-2xl`) 图标容器 (背景 `softPeach` 等浅色)，右侧数据排版。
- **信息/提示卡片**: 背景使用 `softPeach/30` 或 `pastelYellow/40`，带有同色系 `1px` 边框，内部左侧放置提示图标。

## 12. 导航组件规范

- **侧边栏导航项**: 
  - 默认: 透明背景，暖灰字，`py-3 px-4`, `rounded-xl`。
  - Hover: 背景 `pastelYellow` (鹅黄)，文字变为深橙色 (`#D98A6C`)。
  - Active: 同 Hover 状态，增加左侧 `3px` 的品牌色指示条（可选）。
- **面包屑/标签页**: 底部使用 `2px` 的 `warmPink` 指示器，带有平滑滑动动画。

## 13. 弹窗 / 抽屉 / 提示组件规范

- **Modal 弹窗**: 
  - 遮罩层: `rgba(253, 251, 247, 0.6)` 配合 `backdrop-blur-sm` (毛玻璃效果，保持温馨感)。
  - 容器: 最大宽度 `480px`，`rounded-3xl`，无生硬黑边。
- **Toast 提示**: 顶部居中或右下角弹出，纯白背景，左侧带状态色图标，大圆角，暖阴影。
- **Tooltip**: 背景使用 `warmText` (暖灰)，白字，`rounded-lg`，带有小三角箭头。

## 14. 状态规范


| 状态                | 视觉表现                                 |
| ----------------- | ------------------------------------ |
| **默认 (Default)**  | 元素基础样式。                              |
| **悬停 (Hover)**    | 阴影加深，背景色微亮，或 `translateY(-2px)`。     |
| **激活 (Active)**   | 元素轻微缩小 `scale(0.98)`，阴影收缩。           |
| **禁用 (Disabled)** | 透明度降至 `50%`，鼠标指针变为 `not-allowed`。    |
| **加载 (Loading)**  | 按钮内嵌旋转 Spinner，文本变灰，禁用点击。            |
| **错误 (Error)**    | 边框变为 `#FCA5A5`，底部出现 `12px` 红色错误提示文本。 |
| **成功 (Success)**  | 边框变为 `#86EFAC`，输入框右侧出现绿色对勾图标。        |


## 15. 响应式规则

- **断点定义**: `sm` (640px), `md` (768px), `lg` (1024px), `xl` (1280px)。
- **侧边栏处理**: `< 1024px` 时隐藏左侧侧边栏，改为顶部汉堡菜单或底部 Tab 导航。
- **栅格降级**: 
  - `grid-cols-3` 在 `< 768px` 时降级为 `grid-cols-1`。
  - `grid-cols-2` 在 `< 640px` 时降级为 `grid-cols-1`。
- **间距压缩**: 移动端页面边距由 `p-8` 缩减为 `p-4`。

## **16. 后续新增页面必须遵守的规则**

1. **色彩禁忌**：全局禁止使用纯黑 (`#000000`) 和纯白 (`#FFFFFF` 作为背景时)，必须使用 `warmText` 和 `cream` 替代。
2. **形状禁忌**：禁止使用 `border-radius < 8px` 的元素，所有按钮和卡片必须保持圆润。
3. **动效强制**：所有可交互元素（按钮、卡片、导航项）必须添加 `transition-all duration-300`。
4. **文案温度**：错误提示禁止使用 "Error"、"Failed"，应使用 "小助手遇到了一点问题"、"网络连接似乎睡着了" 等拟人化文案。
5. **隐私优先**：任何涉及 API Key 或截图区域的页面，必须在顶部或侧边栏保留“隐私小贴士”组件。

## **附录 A：组件展示页 (Components Preview) 规划**

为了验证设计系统的完整性，需在 `/components-preview` 路由下创建一个纯展示页面。该页面不包含业务逻辑，仅用于开发时的视觉走查。

### **页面区块划分：**

1. **色彩与排版看板 (Color & Typography Board)**
  - 展示所有品牌色块及 HEX 值。
  - 展示 H1-H3 及正文、标签的实际渲染效果。
2. **按钮矩阵 (Button Matrix)**
  - 横向排列：Primary, Secondary, Ghost, Danger。
  - 纵向排列：Default, Hover (强制触发), Active, Disabled, Loading。
3. **表单实验室 (Form Lab)**
  - 包含 Input, Select, Textarea, Switch 的正常态、Focus 态、Error 态（带红色边框和提示语）。
4. **卡片画廊 (Card Gallery)**
  - 展示基础白卡、带图标的数据统计卡、带浅黄背景的隐私提示卡。
5. **反馈与覆盖层 (Feedback & Overlays)**
  - 提供按钮触发 Toast 弹出。
  - 提供按钮触发 Modal 弹窗（展示毛玻璃遮罩效果）。
  - 展示 Skeleton 骨架屏（使用 `softPeach` 颜色的闪烁动画）。
6. **真实业务场景模拟 (Real-world Sections)**
  - **Hero 区块**: 模拟首页的“下午好，今天也要开心直播呀！”及状态指示灯。
  - **设置面板**: 模拟 API 配置与截图区域坐标输入的 Grid 布局。
