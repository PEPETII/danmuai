# DanmuAI 2026-05-27

建议 Git tag：`v2026.05.27`

## 亮点

- **仅保留普通弹幕模式**：固定识图间隔 + 每批条数；移除实时模式与相关 Web 配置项，行为更稳定、文档与代码一致。
- **公式化弹幕库**：独立侧栏页管理内置/自定义短句、最小同屏补足；新 API `/api/danmu-pool/*`。
- **人格工坊 +7**：新增傲娇、腹黑、中二、治愈、毒舌、元气、社恐等内置人格（共 16 种内置，需在激活列表勾选后参与随机）。
- **前台活动感知**：根据当前前台窗口推断写代码/游戏/浏览等活动，写入记忆提示（`app/window_info.py`、`app/memory/activity*`）。
- **架构收敛**：运行态与调度迁入 `app/application/`；Boundary Guard 与验收脚本 `scripts/boundary_guard.py`、`scripts/run_acceptance_gates.py`。
- **Web 控制台**：教程（飞书外链）、问题反馈（QQ 群二维码）；移除遗留 Qt 主窗启动方式。

## 用户可见变更

### 新增

- Web 侧栏 **公式化弹幕库**：内置/自定义公式化短句、最小同屏条数、自定义句增删。
- API：`GET/PUT /api/danmu-pool/meta|settings`、`GET/POST/DELETE /api/danmu-pool/custom`。
- 人格工坊内置人格：**傲娇型、腹黑型、中二型、治愈型、毒舌型、元气型、社恐型**。
- Web **教程**、**问题反馈** 页。
- 小米 **MiMo** 服务商预设与模型目录（`mimo-v2.5` 等）；助手设置视觉模型选择器优化。
- 前台窗口活动摘要（用于 AI 场景理解，非弹幕文本记忆）。

### 移除

- 遗留 Qt 主窗（`ui/`）及 `--qt-ui`、`--legacy-ui`、`DANMU_QT_UI`、`DANMU_WEB_CONSOLE=0`（启动将 `exit 2` 并提示迁移）。
- **实时弹幕模式**（`danmu_display_mode=realtime`）及 Web 表单项：`reply_scene_count`、`reply_filler_count`、`screenshot_interval`、`freq_mode`、`capture_mode`、`freshness`、`drop_stale`、`scene_probe_size`、`memory_clear_policy`。
- 助手设置中的「内置中文短句库」「最小同屏条数」（已迁至公式化弹幕库）。

### 变更

- 公式化补足：`min_on_screen` 在内置库或自定义库任一开启时生效。
- `PUT /api/config` 不再包含 `danmu_pool_enabled` / `min_on_screen`（请用 `/api/danmu-pool/settings`）。
- 默认始终启动 Web 控制台 + pywebview + Qt Overlay/托盘。
- 开源文档治理：正式文档集 + `docs/archive/`；新增 `CONTRIBUTING_ARCHITECTURE`、`MAIN_PIPELINE`、`RUNTIME_STATE`、`BOUNDARY_GUARD`、`PACKAGING_WINDOWS` 等。

## 升级说明

1. **不要用** `--qt-ui` / `DANMU_WEB_CONSOLE=0`；请直接 `python main.py`。
2. 若曾使用 **实时模式**，配置会在启动/Web 保存时按普通模式处理；请在助手设置确认 `normal_recognition_interval_sec`、`normal_reply_count`。
3. **公式化弹幕库** 请到新侧栏页开关与条数；旧配置键若仍写在 `config.db` 中，以新 API 为准。
4. 新人格默认**未勾选激活**，请到 **人格工坊 → 激活人格** 勾选并保存。
5. Windows exe：见 [PACKAGING_WINDOWS.md](../PACKAGING_WINDOWS.md)；需 WebView2 或 `--web-browser`。
