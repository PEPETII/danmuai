# 工单 W-023 — 助手设置「恢复默认」

## 工单 ID

W-023

## 工单标题

助手设置界面新增「恢复默认」按钮（仅改表单，不自动保存）

## 背景

当前「助手设置」仅有「保存配置」。用户误改多项配置后恢复成本高，需要快速将表单恢复为工厂默认值，且须避免误覆盖 API Key 与自定义模型/人格。

## 目标

- 「保存配置」左侧出现次要按钮「恢复默认」
- 点击后弹窗：仅恢复当前界面 / 恢复所有设置 / 取消
- 恢复后 toast 提示须手动保存；刷新未保存时配置不变

## 依赖项

无

## 允许修改的区域

- `web/static/index.html`
- `web/static/app.js`
- `app/config_defaults.py`
- `app/application/config_service.py`
- `app/web_console.py`
- `tests/test_web_console.py`
- `docs/WEB_CONSOLE.md`、`docs/当前仓库状态.md`、`docs/工单列表.md`

## 禁止修改的区域

- `main.py` 主链路
- `app/overlay.py`、`app/danmu_engine.py`、`app/ai_client.py`
- `app/web_api/routes.py`（本工单 `/api/config/defaults` 注册在 `web_console.py`）
- reply 调度、打包脚本、依赖文件

## 需求

1. 新增「恢复默认」次要按钮（位于「保存配置」左侧）
2. 确认弹窗含三个选项：仅当前 Tab / 全部 / 取消
3. 仅当前 Tab：按 Tab 映射恢复字段；识图区域 Tab 无表单项时提示
4. 全部：恢复所有 `WEB_CONFIG_KEYS` 对应表单字段 + checkbox
5. 不恢复 API Key、自定义模型、人格、识图区域持久化、公告/统计
6. 不自动 POST；用户须点「保存配置」才持久化
7. `GET /api/config/defaults` 为唯一默认值来源
8. 关键逻辑中文注释

## 非目标

- 不清空 API Key
- 不删除自定义模型/人格
- 不恢复识图区域（仍用「恢复全屏识图」）
- 不引入新依赖或架构分层

## 验收标准

- [x] 按钮位置与样式符合次要操作
- [x] 弹窗三选项可用
- [x] 当前 Tab / 全部恢复范围正确
- [x] API Key 不参与恢复
- [x] 恢复后不自动保存
- [x] pytest + boundary_guard 通过

## 手动验证步骤

1. 改助手设置若干项 → 恢复默认 → 仅当前 Tab → 确认仅当前分组回默认
2. 恢复全部 → 确认多 Tab 字段回默认，自定义模型列表不变
3. 不保存刷新 → 配置仍为旧值
4. 保存后刷新 → 新默认值持久化
5. 识图区域 Tab → 仅当前界面 → 提示无可恢复项

## 风险点

- 禁止调用 `applyProviderPreset`（会清空 api_key 输入）
- `screen_index` 默认 `"0"` 与首装一致
