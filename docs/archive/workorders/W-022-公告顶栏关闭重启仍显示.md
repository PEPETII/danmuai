# W-022：修复温馨控制台顶栏公告关闭后重启仍显示

## 工单 ID

W-022

## 工单标题

修复温馨控制台顶栏公告关闭后重启仍显示

## 背景

W-008 顶栏简略条关闭状态仅写入 `localStorage`（`danmu_announcements_overview_banner_dismissed_id`）。与 ISSUE-008（侧栏红点）同类：pywebview / WebView2 完全退出后 `localStorage` 常不保留。W-ANNOUNCE-BADGE-001 已将侧栏已读迁入 `config.db`；顶栏关闭尚未迁入。

## 目标

1. 用户点顶栏「×」后，完全退出应用再启动，同一公告仍为列表首条时顶栏不再显示。
2. Supabase 发布新公告（新 `id` 成为首条）后顶栏再次显示。
3. 与侧栏未读红点语义独立（关顶栏不自动消红点）。

## 依赖项

- W-008（顶栏 UI）、W-ANNOUNCE-BADGE-001（`announcements-read-state` API）
- 可选：`web/static/supabase-config.js` 用于手动验收

## 允许修改的区域

- `app/web_api/routes.py`
- `web/static/app.js`
- `tests/test_web_console.py`
- `docs/WEB_CONSOLE.md`
- `docs/当前仓库状态.md`

- `docs/templates/已知问题记录/ISSUE-016-公告顶栏关闭重启仍显示.md`

## 禁止修改的区域

- `main.py`
- 弹幕主链路（`app/overlay.py`、`app/danmu_engine.py` 等）
- Supabase 迁移
- `requirements.txt`、锁文件

## 需求

1. `announcements_read_state` 增加 `overviewBannerDismissedId`（UUID 或空字符串）。
2. `GET/PUT /api/announcements-read-state` 读写该字段至 `config.db`。
3. `app.js` 启动时合并 remote + localStorage；点 × 时双写并 `PUT`。
4. 扩展 pytest 覆盖默认、往返、非法 UUID。

## 非目标

- 不改顶栏 30 字截断逻辑
- 不把「点 ×」并入 `readIds`
- 不修 ISSUE-007（Supabase 外网不可用）

## 验收标准

- [x] 有已发布公告且未关闭时顶栏显示；点 × 后隐藏（代码已实现，手动待签字）
- [x] 托盘完全退出再启动，同一 `id` 首条时顶栏不显示
- [x] 新公告成为首条后顶栏再现
- [x] 关闭顶栏不自动消除侧栏红点

## 手动验证步骤

1. 配置 `supabase-config.js`，确保有一条已发布公告。
2. 启动应用 → 温馨控制台 → 确认顶栏显示 → 点 × → 顶栏消失。
3. 托盘完全退出 → 再启动 → 顶栏仍不出现。
4.（可选）Dashboard 新发公告 → 顶栏再现；未进公告页时红点仍可用。

## 风险点

- 仅存储单个 dismissed id；同 id 改正文不会再次显示顶栏（W-008 已知限制）。
- 旧版仅 localStorage 的用户首次启动会经合并写入 `config.db`。
