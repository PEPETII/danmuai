# W-008：温馨控制台顶部公告简略条

## 背景

已有 Supabase 公告列表与侧栏未读红点（`ANNOUNCEMENTS_LAST_SEEN_KEY`）。用户希望在「温馨控制台」顶部快速看到最新公告摘要，可一键关闭；关闭后同一则不再显示，直到发布新公告。

## 目标

1. `#page-overview` 顶部展示最新公告「标题：正文」前 30 字（超长加 `…`）。
2. 用户点叉后写入 `localStorage`（按公告 `id`），刷新后仍隐藏。
3. 列表首条 `id` 变化（新公告/置顶）时顶栏再次出现。
4. 与侧栏未读红点逻辑分离（叉掉不写 `last_seen_at`）。

## 依赖项

- Supabase 公告迁移与 `web/static/supabase-config.js`（可选；未配置时顶栏静默不显示）

## 允许修改的区域

- `web/static/index.html`
- `web/static/app.js`
- `web/static/warm-tokens.css`
- `tests/test_bundle_paths.py`
- `docs/WEB_CONSOLE.md`
- `docs/工单列表.md`、`docs/当前仓库状态.md`
- `docs/templates/工单/`、`docs/templates/Codex完成报告/`

## 禁止修改的区域

- `app/`、`main.py`
- `supabase/` 迁移、`requirements.txt`

## 需求

1. 在 `#page-overview` 的 `<header>` 之前插入可关闭顶栏 DOM。
2. `buildAnnouncementSnippet`：`title + "：" + body`，`Array.from` 截 30 字 + `…`。
3. `danmu_announcements_overview_banner_dismissed_id` 存储已关闭的公告 id。
4. `refreshAnnouncementsUnreadBadge` 单次 `listAnnouncements` 同时更新红点与顶栏。
5. 未配置 Supabase 或请求失败时顶栏静默隐藏。

## 非目标

- 全站顶栏、点击跳转公告页
- 修改 `last_seen_at` / 红点语义
- 后端代理 Supabase、ISSUE-007 全局静默

## 验收标准

- [x] 有已发布公告且未叉掉时，控制台顶栏显示截断文案（代码已实现，手动待签字）
- [x] 点叉后 F5 仍隐藏同一 `id`
- [x] 新公告成为列表首条后顶栏再现
- [x] 未配置/拉取失败时顶栏不出现
- [x] 侧栏红点行为与改前一致
- [x] `pytest tests/test_bundle_paths.py tests/test_supabase_static.py -q` 通过

## 手动验证步骤

1. 配置 `supabase-config.js`，打开温馨控制台
2. 确认顶栏与公告页最新一条一致（截断）
3. 叉掉 → F5 → 仍隐藏
4. Dashboard 新增公告 → 顶栏再现
5. 未进公告页时侧栏红点仍可用（若有未读）

## 风险点

- 同 `id` 仅改正文不会再次显示顶栏（按 id 关闭，可接受）
