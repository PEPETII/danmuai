# W-025：新增测试1–4 内置人格并置顶人格工坊

## 工单 ID

W-025

## 工单标题

新增测试1–4 内置人格并置顶人格工坊

## 背景

需四套试验性「五人弹幕」人格（测试1 为通用直播间风格；测试2–4 为竞技/唠嗑/阴阳梗主题），在人格工坊列表置顶；移除旧单字内置「测试」。

## 目标

人格工坊下拉与激活列表首四项为测试1–4；运行时 `get_prompt` 有效；旧「测试」从内置与激活列表清理。

## 依赖项

- W-PROMPT-001（全局输出契约 + 短句正式人格）

## 允许修改的区域

- `app/personae.py`
- `app/translations.py`
- `tests/test_reply_contract.py`
- `tests/test_web_persona_api.py`
- `docs/`（工单、完成报告、docs/test）

## 禁止修改的区域

- `main.py`、`web/static/`、`app/web_api/`、`app/ai_client.py`
- 16 个正式内置人格文案
- `requirements.txt`、Boundary Guard 登记表

## 需求

1. 新增 `BUILTIN_PERSONAE` 测试1–4（`system_zh` 仅人格段）。
2. `BUILTIN_PERSONA_PINNED_FIRST` + `PersonaManager.list()` 置顶。
3. 移除「测试」；`_REMOVED_PERSONAE` 含「测试」。
4. 翻译键 `persona.test1`–`test4`。
5. 测试与 `docs/test/` 登记文档。

## 非目标

- 不修改全局 `build_normal_reply_contract_*`
- 不默认激活测试1–4

## 验收标准

- [x] 列表首四项为测试1–4
- [x] 无内置「测试」
- [x] pytest 全绿

## 手动验证步骤

1. `python main.py` → 人格工坊 → 下拉顺序与测试1 系统提示。
2. 勾选测试1 保存激活 → 开始弹幕（主观看五人分化）。

## 风险点

- 旧用户 `active_personae` 含「测试」会被过滤，需手动补勾。
