## 1. 修改摘要

将 pywebview 父子进程握手从单次 `put(True)` 改为二阶段：`created`（窗口已建、即将 `webview.start()`）→ `loaded`（`on_loaded` 内 `show()` 后）。父进程在总超时 20s 内循环读队列，仅在收到 `loaded` 后设 `_started`；`start()` 失败、子进程提前退出或 loaded 超时则终止子进程并 `_fallback_to_system_browser()`。保留 ISSUE-010 约束：`hidden=True`，禁止在 `start()` 前 `show()`。

- SPA / `#settings` 若长期不触发 `loaded`，会在 `_LOAD_TIMEOUT_SEC`（打包 15s）后回退浏览器，优于静默无窗。
- 勿在 `webview.start()` 前 `show()`（ISSUE-010）。
- 勿将 `loaded` 作为唯一首包（ISSUE-010 曾 20s 超时）。

**握手协议（`app/webview_shell.py`）**

```text
create_window(hidden=True)
ready_queue.put("created")
webview.start(...)
on_loaded -> window.show(); ready_queue.put("loaded")
父进程: created -> 等待 loaded -> _started=True
失败/超时/子进程退出 -> fallback browser
```
