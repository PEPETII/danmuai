## 7. 风险与注意事项

- **032**：`stop()` 与 `start()` 对称依赖 `_sync_mic_service()`；仅 `quit()` 无条件 stop 硬件。配置保存路径仍走 orchestrator，设备 ID 变化时 `MicCaptureService.start()` 仍会重启流（允许行为）。
- **033**：runtime flush 失败时 `stop()` 会抛 `OperationalError`，session 不会写入；与 BUG-010「失败不清 clock」语义一致。
- **034**：池补齐 item 初始在屏外，循环内 visible deficit 通常不变；本地 `limit` 与原先「最多 min(deficit,8) 条」语义等价。
- **035**：跨 tick 的 50ms 主线程合并未做（非本票范围）；同线程突发仍由 hub `_pending` 合并。
- **RISK-017**：30min Windows audio soak 仍须负责人另验，本票未覆盖。
